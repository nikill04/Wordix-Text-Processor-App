An **event** is:
> **Any action done by the user in the browser**
Examples:
* Typing in a textarea
* Clicking a button
* Moving the mouse
* Pressing a key

When an event happens, the browser **creates an object** that describes **what happened**.
That object is called an **event object**.

## 2Ô∏è‚É£ What does React do with events?

React:
1. Listens for browser events
2. Wraps them inside a **SyntheticEvent**
3. Sends that event object to your function automatically

You do **NOT create the event**
You only **receive it**

## 3Ô∏è‚É£ How does the event reach your function?
When you write:
<textarea onChange={handleChange} />

You are saying:
> ‚ÄúReact, when something changes in this textarea, call `handleChange` and give it the event.‚Äù

So React internally does:
const handleOnChange = (event) => {
  ...
}

You could name it anything:
(e) => {}
(evt) => {}

## 4Ô∏è‚É£ What is inside the **event object**?

The event object contains **details about the action**.

Important parts:

| Property          | Meaning                           |
| ----------------- | --------------------------------- |
| `event.type`      | Type of event (`change`, `click`) |
| `event.target`    | Element that caused the event     |
| `event.timeStamp` | When it happened                  |

## 5Ô∏è‚É£ What is `event.target`?
> **`event.target` is the HTML element that triggered the event**

If user types in a textarea:
<textarea>hello</textarea>

Then:
event.target === textarea element

## 6Ô∏è‚É£ What is `event.target.value`?
Form elements like:
* `input`
* `textarea`
* `select`

store user input in `.value`

So: event.target.value   means:

> ‚ÄúGive me the current text inside this textarea‚Äù
Example:
* User types: `hello`
* `event.target.value` ‚Üí `"hello"`

## 8Ô∏è‚É£ How data flows when user types (GENERAL FLOW)
User types
   ‚Üì
Browser fires "change" event
   ‚Üì
Event object is created
   ‚Üì
React passes event to handler
   ‚Üì
You read event.target.value
   ‚Üì
You decide what to do with it

Same logic applies to:
* `<input>`
* `<select>`
* `<checkbox>`
* `<radio>`

Only difference:
* property might be `.checked` instead of `.value`
